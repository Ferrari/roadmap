io.js 光明的未來

社群驅動的路線圖

社群擁有，社群管理

io.js 是 node.js 的替代品，可互換，並且幾乎兼容 npm 中的所有模組。

io.js 是一個社群驅動的開源專案，是由 node.js 的主要貢獻者發起的，並且在最初的幾個月間吸引了比 node.js 出現以來更多的活躍開發者。

從 io.js 能得到什麼?

  1. 穩定性
  2. 透明性 (調試和追蹤)
  3. 更好的 Streams
  4. 長期支持
  5. 擁抱標準
  6. 本地化

相關列表編輯自 roadmap 項目，歡迎貢獻者!

穩定性策略

在各種語言中我們擁有最大的模組生態系統，我們不能毀了它。

  • 不會移除 JS API.
  • 語義化版本
    • 添加 API 會增加 minor 版本
    • 如果 V8 的 C++ API 的變化能被 nan 兼容，則增加 minor 版本，否則增加 major 版本。

來自路線圖的 穩定性策略 章節，歡迎貢獻者!

Channels

Release 任何被標記版本都會編譯
必須穩定且可用於線上環境
Canary 每晚編譯，包含 Canary V8 及其他新特性
NG 可使用下一代 ES6/7 核心 API.


來自 路線圖，歡迎貢獻者!

NG (Next Generation)

  • 現代 JavaScript 應用的一個新平台
  • 將會一直保持向下兼容
  • 新的 API 將會以 ES6/7 為中心，但會兼容原來的模組和 API
  • 將會把對於現在平台的改善和對未來 API 的工作區分開

NG 討論還在持續進行中 NG 程式庫

除錯和追蹤

應該在核心使用大量且多樣化的除錯和追蹤工具，而不是提供高層級的抽象，如 domains

  • 集成 trace_event (V8 和 Chrome 的新 trace API).
  • 添加文件和宣傳這種調試生態系統
  • 統一系統追蹤終端 (DTrace, LTTng, etc)

由追蹤工作組領導，歡迎貢獻者!

Streams

  • Streams 工作組 已經接管了 readable-stream，它已經成為了 io.js 官方 stream 模塊
  • 解決所有現存的兼容問題
  • 簡化生成 stream，避免造成用戶使用錯誤
  • 探索 WHATWG Streams 接口，識別兼容性問題
  • 提升 stream 性能

由 Streams 工作組領導，歡迎貢獻者!

長期支持

  • 新的發佈將會和穩定及所支持的依賴一起發佈
  • 只要有人一直積極的貢獻，老版本依然會發佈 patch 版本。
  • 如果依賴庫（如 V8）的維護者停止支持老版本，在以前的發佈中出現過，io.js 將會對此負責

從穩定性策略總結，歡迎貢獻!

本地化

  • 減少引入 icu 的影響 (二進制和內存佔用空間).
  • 推進充滿活力的本地化社區的發展
    • 自主性的語言社群 (到現在已經有 30+)
    • 宣傳以增加成員人數 (到現在已經有 160+ 人)
    • 優化工具，支持翻譯站點，API 文件，安裝器，PPT

如果你想加入本地化社群請到 這裡留言 :)
